## Часть 1: Подготовка и основы

#### Задача

Скрипт-обработчик логов с вложенными циклами и оптимизация их. 

#### Процесс работы

Логи генерируем произвольно. В каждый лог входит:

- id лога
- Дата и время лога
- Уровень важности сообщения
- Сообщение

Уровни важности сообщения:
- INFO
- WARNING
- ERROR
- DEBUG
- CRITICAL

Сообщение состоит из некоторого текста, который может включать:
- Логин пользователя
- IP
- Имя файла
- Время
- Процент

Имена всех пользователей храним в одном массиве.

Сформулируем задачу так: распределить все логи уровня ERROR по соответствующим пользователям. 

_1 вариант - Вложенные циклы:_

Проходимся по списку пользователей и для каждого пользователя проходимся по списку логов.

> Проблема: мы проходимся по списку логов столько раз, сколько у нас пользователей. Если логов 50 000, а категорий 5, мы сделаем **250 000 операций сравнения**.

_2 вариант - Оптимизация:_

Сразу проходимся по списку логов, уже потом выявляя пользователя.
> Количество операций сравнения снижается **экспоненциально**.

#### Структура подпроекта
- /1_log/generate_log.py - генерация 50 тысяч произвольных логов
- /1_log/main.py - реализация функции с вложенными циклами, оптимизированной функции и сравнение
